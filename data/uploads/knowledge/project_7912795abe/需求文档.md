# 智能模型预测平台 - 需求文档 v2.0

## 1. 项目概述

### 1.1 项目背景
在各类工程项目中，数据驱动的决策和优化越来越重要。传统的参数设置和问题解决方式往往依赖人工经验，存在效率低、准确性不足、优化周期长等问题。本项目旨在构建一个通用的Web智能模型预测平台，支持用户上传自己的数据、训练自定义模型、管理项目知识库，并通过AI助手提供智能问答服务，适用于各类工程领域的数据分析和优化需求。

### 1.2 项目目标
- 构建灵活的数据分析平台，支持可视化展示和交互式探索
- 实现端到端的机器学习工作流，支持用户自定义模型训练和预测
- 建立项目化的知识库管理体系，支持多项目文档管理和检索
- 开发AI智能助手，提供自由对话和基于项目知识库的RAG问答

### 1.3 项目定位
面向工程师、研究人员和技术人员的通用智能模型预测平台，通过数据驱动和AI技术提升工程项目的分析效率和决策质量。平台不限定特定领域，可应用于制造、化工、材料、能源、建筑等各类工程场景。

### 1.4 应用场景示例
- **制造工程**：工艺参数优化、质量预测、设备故障诊断
- **材料工程**：材料性能预测、配方优化、工艺参数设计
- **化工工程**：反应参数优化、产物收率预测、过程控制
- **能源工程**：能源消耗预测、设备效率优化、负荷预测
- **建筑工程**：结构性能预测、施工参数优化、质量评估

## 2. 功能需求

### 2.1 数据分析模块

#### 2.1.1 功能描述
提供数据可视化分析功能，用户可以创建和管理分析卡片，展示关键指标和图表。

#### 2.1.2 功能要求
- **卡片预览**：首页展示预设的分析卡片，包括：
  - 特征相关性热力图
  - 数据分布概览
  - 关键指标趋势图
  - 模型性能对比

- **卡片管理**：
  - 创建新的分析卡片
  - 编辑卡片标题、图注和分析文本
  - 上传分析图片（PNG/JPG/JPEG/WEBP）
  - 更新卡片内容
  - 删除卡片

- **可视化展示**：
  - 支持多种图表类型（折线图、柱状图、散点图、热力图等）
  - 图表交互（缩放、筛选、悬停提示）
  - 图片预览和下载

#### 2.1.3 卡片数据结构
```json
{
  "id": "card_uuid",
  "title": "卡片标题",
  "caption": "图注说明",
  "image_url": "/static/assets/analysis/xxx.png",
  "analysis": "分析文本内容"
}
```

**数据要求**：
- 第一行为列名（特征名）
- 数据行数建议≥100条
- 数值型特征：连续值或离散值
- 分类特征：建议进行编码（如：类型A=1, 类型B=2, 类型C=3）
- 不允许有空值或缺失值

#### 2.2.3 特征选择

**功能描述**：
用户选择哪些列作为输入特征（X），哪些列作为输出目标（Y）。

**交互要求**：
- 自动识别所有列名
- 支持多选输入特征
- 支持多选输出目标
- 显示特征类型（数值型/分类型）
- 提供智能推荐（基于特征重要性）

#### 2.2.4 数据处理

**自动处理流程**：
1. **数据验证**：检查数据格式、缺失值、异常值
2. **特征工程**：
   - 数值型特征：标准化（StandardScaler）
   - 分类型特征：独热编码（OneHotEncoder）
3. **特征筛选**：基于特征重要性筛选冗余特征
   - 使用随机森林计算特征重要性
   - 保留重要性前N个特征（用户可配置）
   - 生成特征重要性排序图

**实时展示**：
- 特征重要性柱状图
- 特征相关性热力图
- 数据分布直方图
- 处理前后数据对比

#### 2.2.5 模型选择

**内置模型库**：

**机器学习模型**：
- 线性回归（Linear Regression）
- 岭回归（Ridge Regression）
- Lasso回归（Lasso Regression）
- 决策树（Decision Tree）
- 随机森林（Random Forest）
- 梯度提升树（Gradient Boosting）
- XGBoost
- LightGBM
- 支持向量回归（SVR）
- K近邻（KNN）

**深度学习模型**：
- 多层感知机（MLP）
- 深度神经网络（DNN）
- 卷积神经网络（CNN，适用于序列数据）
- LSTM（适用于时序数据）
- Transformer（适用于复杂关系）

**模型参数配置**：
- 每个模型提供可调参数界面
- 提供默认参数和推荐参数
- 支持参数搜索（网格搜索、随机搜索）
- 显示参数说明和取值范围

#### 2.2.6 模型训练

**训练流程**：
1. **数据集自动划分**：系统自动将上传的训练集划分为训练集（80%）、验证集（10%）、测试集（10%），用户可自定义划分比例
2. 模型初始化
3. 模型训练（实时显示进度）
4. 模型评估（在测试集上评估）
5. 结果可视化

**实时交互展示**：
- **训练进度**：进度条、已用时间、预计剩余时间
- **损失曲线**：训练损失和验证损失随epoch变化
- **评估指标**：R²、MAE、MSE、RMSE等
- **预测结果对比**：实际值vs预测值散点图
- **残差分析**：残差分布图

**训练参数保存**：
- 模型架构和权重
- 超参数配置
- 数据预处理参数（scaler、encoder）
- 特征列表和目标列表
- 训练日志和评估指标
- 训练时间戳

#### 2.2.7 模型保存与卡片生成

**模型保存**：
- 训练完成后，用户可以选择保存模型
- 用户为模型命名（如："工艺参数优化模型_v1"）
- 保存后自动生成模型卡片，显示在"我的模型"列表中
- 模型保存到独立的UUID文件夹

**模型卡片内容**：
- 模型名称和描述
- 训练时间戳
- 评估指标（R²、MAE、MSE、RMSE等）
- 输入特征列表
- 输出目标列表
- 模型类型和超参数
- 训练数据预览

**文件夹结构**：
```
models/{model_uuid}/
├── model/              # 模型文件
│   ├── model.pkl       # scikit-learn模型

│   ├── features.json
│   └── targets.json
├── data/               # 数据文件

├── results/            # 结果文件
│   ├── metrics.json

    └── training.log
```

**项目创建**：
- 创建新项目
- 项目命名

- 项目名称、描述、文档数量

**项目切换**：
- 在不同项目间切换

- 支持PDF文件上传

### 2.4 智能助手模块

#### 2.4.1 功能描述
提供AI智能对话服务，支持自由聊天和基于项目知识库的RAG问答。

#### 2.4.2 交互模式

**自由聊天模式**：
- 解答技术问题
- 提供专业建议和指导
- 支持多轮对话
- 不依赖知识库

**RAG问答模式**：
- 选择项目知识库
- 基于知识库检索相关内容
- 结合检索结果生成回答
- 显示引用来源（文档名、内容片段）
- 支持多项目知识库联合检索

#### 2.4.3 RAG引用

**引用格式**：
```
根据《[文档名称]》的内容，[回答内容]。

引用来源：
- [文档名称]：[内容片段]
```

**引用要求**：
- 准确标注引用来源
- 显示文档所属项目
- 提供原文链接
- 支持点击跳转

#### 2.4.4 技术要求
- 支持流式输出（SSE）
- 集成大语言模型（OpenAI兼容API）
- 支持历史对话记录
- 支持多项目知识库选择
- 提供专业、准确的回答
- 引用准确性验证

## 3. 非功能需求

### 3.1 性能要求
- 模型训练：支持异步处理，不阻塞用户操作
- 预测响应时间 < 2秒
- 知识检索响应时间 < 1秒
- 支持并发用户访问（建议10+）
- 页面加载时间 < 3秒
- 支持大文件上传（建议100MB+）

### 3.2 可用性要求
- 界面简洁友好，操作直观
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 提供清晰的操作指引和错误提示
- 支持响应式布局
- 提供数据规范示例和模板
- 提供模型训练向导

### 3.3 可扩展性要求
- 支持模型类型扩展（插件式架构）

## 5. 数据需求

- 分类型特征：建议进行编码

- 每个模型独立UUID文件夹

## 6. 接口设计

### 6.1 数据分析接口

#### 6.1.1 获取分析卡片列表
- **路径**: GET /api/analysis/cards
- **描述**: 获取所有分析卡片
- **返回**: 卡片列表

#### 6.1.2 创建分析卡片
- **路径**: POST /api/analysis/cards
- **描述**: 创建新的分析卡片
- **输入**: title, caption, image_url, analysis
- **返回**: 创建的卡片

#### 6.1.3 更新分析卡片
- **路径**: PUT /api/analysis/cards/{card_id}
- **描述**: 更新分析卡片
- **输入**: title, caption, image_url, analysis
- **返回**: 更新后的卡片

#### 6.1.4 删除分析卡片
- **路径**: DELETE /api/analysis/cards/{card_id}
- **描述**: 删除分析卡片
- **返回**: 成功/失败

#### 6.1.5 上传分析图片
- **路径**: POST /api/analysis/upload-image
- **描述**: 上传分析图片
- **输入**: 图片文件（PNG/JPG/JPEG/WEBP）
- **返回**: 图片URL

### 6.2 模型训练与预测接口

#### 6.2.1 上传训练数据
- **路径**: POST /api/training/upload-data
- **描述**: 上传训练数据文件
- **输入**: 文件（CSV/Excel）
- **返回**: task_uuid, 列名列表, 数据预览

#### 6.2.2 获取可用模型列表
- **路径**: GET /api/training/models
- **描述**: 获取所有可用模型
- **返回**: 模型列表（名称、类型、描述、参数）

#### 6.2.3 获取模型参数配置
- **路径**: GET /api/training/models/{model_name}/config
- **描述**: 获取模型的参数配置
- **返回**: 参数列表（名称、类型、默认值、范围、描述）

#### 6.2.4 开始训练
- **路径**: POST /api/training/start
- **描述**: 开始模型训练
- **输入**: task_uuid, model_name, hyperparams, feature_columns, target_columns
- **返回**: training_id, 状态

#### 6.2.5 获取训练进度
- **路径**: GET /api/training/progress/{training_id}
- **描述**: 获取训练进度（WebSocket）
- **返回**: 进度信息（epoch, loss, metrics, logs）

#### 6.2.6 获取训练结果
- **路径**: GET /api/training/results/{training_id}
- **描述**: 获取训练结果
- **返回**: 评估指标、预测结果、图表URL

#### 6.2.7 保存模型
- **路径**: POST /api/training/save-model
- **描述**: 保存训练好的模型
- **输入**: training_id, model_name
- **返回**: model_uuid

#### 6.2.8 获取已保存模型列表
- **路径**: GET /api/models
- **描述**: 获取所有已保存的模型
- **返回**: 模型列表（名称、创建时间、评估指标）

#### 6.2.9 获取模型详情
- **路径**: GET /api/models/{model_uuid}
- **描述**: 获取模型详情
- **返回**: 模型配置、训练信息、评估指标

#### 6.2.10 删除模型
- **路径**: DELETE /api/models/{model_uuid}
- **描述**: 删除已保存的模型
- **返回**: 成功/失败

#### 6.2.11 使用模型预测
- **路径**: POST /api/models/{model_uuid}/predict
- **描述**: 使用模型进行预测
- **输入**: 输入参数（根据模型的输入特征）
- **返回**: 预测结果

#### 6.2.12 批量预测
- **路径**: POST /api/models/{model_uuid}/batch-predict
- **描述**: 批量预测
- **输入**: CSV文件
- **返回**: 预测结果文件

#### 6.2.13 获取任务列表
- **路径**: GET /api/tasks
- **描述**: 获取所有任务
- **返回**: 任务列表（ID、状态、进度、时间）

#### 6.2.14 获取任务详情
- **路径**: GET /api/tasks/{task_uuid}
- **描述**: 获取任务详情
- **返回**: 任务配置、日志、结果

### 6.3 知识库接口

#### 6.3.1 创建项目
- **路径**: POST /api/knowledge/projects
- **描述**: 创建新的知识库项目
- **输入**: name, description, tags
- **返回**: project_uuid

#### 6.3.2 获取项目列表
- **路径**: GET /api/knowledge/projects
- **描述**: 获取所有项目
- **返回**: 项目列表

#### 6.3.3 获取项目详情
- **路径**: GET /api/knowledge/projects/{project_uuid}
- **描述**: 获取项目详情
- **返回**: 项目信息、文档列表

#### 6.3.4 删除项目
- **路径**: DELETE /api/knowledge/projects/{project_uuid}
- **描述**: 删除项目
- **返回**: 成功/失败

#### 6.3.5 上传文档
- **路径**: POST /api/knowledge/projects/{project_uuid}/documents
- **描述**: 上传文档到项目
- **输入**: 文件（PDF/TXT）
- **返回**: 文档信息

#### 6.3.6 获取文档列表
- **路径**: GET /api/knowledge/projects/{project_uuid}/documents
- **描述**: 获取项目下的文档列表
- **返回**: 文档列表

#### 6.3.7 删除文档
- **路径**: DELETE /api/knowledge/projects/{project_uuid}/documents/{doc_id}
- **描述**: 删除文档
- **返回**: 成功/失败

#### 6.3.8 知识检索
- **路径**: POST /api/knowledge/search
- **描述**: 搜索知识库
- **输入**: query, project_uuids（可选，多项目检索）, top_k
- **返回**: 检索结果

### 6.4 智能助手接口

#### 6.4.1 智能对话
- **路径**: POST /api/agent/chat
- **描述**: 智能对话（流式输出）
- **输入**: message, mode（chat/rag）, project_uuid（RAG模式）, history
- **返回**: SSE流式响应

#### 6.4.2 获取对话历史
- **路径**: GET /api/agent/history
- **描述**: 获取对话历史
- **返回**: 历史记录列表

#### 6.4.3 清除对话历史
- **路径**: DELETE /api/agent/history
- **描述**: 清除对话历史
- **返回**: 成功/失败
